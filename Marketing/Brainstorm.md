

# What problems does Gridwell try to solve?

- Hitting the required edge of a window to resize it can be cumbersome. 
- Sometimes the windows have so many icons in the title bar that there is no place to actually grab it.
- Most window management tools either just offer a UI to select grid cells or they visualize the potential new position of a window with an empty frame. Both methods lack the visual feedback of seeing the content of the window reacting to the new size.

## Disclaimer

All code and "art"-work (e.g. app icon) in this project is created 100% by AI.

I have created this projects for two reasons:

1. I wanted to _use_ this tool. All current window management tools _(and there are **a lot** of them!)_ did not do what I wanted, so…

2. I have roughly 40 years of software development under my belt, but I wanted to really see how a full project from zero to release would feel like using AI only. Every line of code in this project is written by Claude Code, with Perplexity.ai as research assistant. I did not write a single line of code myself (aside from the occasional `NSLog()`).

On the first point: it really now is the window management tool I always wanted.

On the second: it turns out that in the hands of someone with extensive software development background, these tools can do quite impressive things. I just would not want this tool wielded by someone who does not know the first thing about software development. Which left the question in my mind of how in the future people are supposed to achieve this software development knowledge when no one is writing code anymore…

But besides that, I also made the experience that coding with AI can be very stressful. When you give an agent a task, it can take quite a while. So what are you going to do in the meantime? They say AI is supposed to _increase productivity_. So of course you're spinning up a second task in a second project. And before you know it, you find yourself jumping around between several projects, not being able to finish a single thought because yet another AI agent needs your attention. 

So I made a decision. If I am going to use this workflow in the future, I will be working on two tasks in parallel at the most. And one of them must not be a software project.

And the time waiting for an agent to finish its current taks I'm going to use to do a little physical exercise in front of my desk :)
