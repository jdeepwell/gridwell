# Competing Products

Here is a comprehensive list of window snapping and window management utilities for macOS, organized by category:

***

## Snap & Shortcut-Based (Mainstream)

These are the most popular tools for drag-to-snap and keyboard shortcut window placement: [youtube](https://www.youtube.com/watch?v=O4g3lLr21iE)

- **Rectangle** – Free and open-source; move and resize windows with keyboard shortcuts or snap zones. The go-to beginner-friendly option [rectangleapp](https://rectangleapp.com)
- **Rectangle Pro** – Paid upgrade with more layouts, custom snap areas, and gap control [macpaw](https://macpaw.com/reviews/best-window-manager-mac)
- **Magnet** – One of the top-paid App Store apps; snap windows via drag, keyboard shortcuts, or menu bar [asoftclick](https://asoftclick.net/best-window-manager-mac/)
- **BetterSnapTool** – $2.99; highly customizable zones, keyboard shortcuts, custom predefined positions, multi-monitor support [maketecheasier](https://www.maketecheasier.com/5-window-managers-for-better-organization/)
- **Moom** – Replaces the macOS green zoom button with a full layout picker; supports up to 61 actions, app-specific layouts, and keyboard shortcuts [macpaw](https://macpaw.com/reviews/best-window-manager-mac)
- **Mosaic** – Drag-based layout tool with preset and custom layouts; shows a layout bar at the top when dragging [macpaw](https://macpaw.com/reviews/best-window-manager-mac)
- **Divvy** – Grid-based window sizer accessed from the menu bar or keyboard shortcut [talk.macpowerusers](https://talk.macpowerusers.com/t/window-snapping-tool/18652)
- **Swish** – Gesture-first window manager using trackpad swipes; no need for keyboard shortcuts [youtube](https://www.youtube.com/watch?v=uakHtSxl6cA)
- **BetterTouchTool (BTT)** – Highly versatile automation tool that includes powerful window snapping and custom gesture triggers [reddit](https://www.reddit.com/r/macapps/comments/1ecgvma/window_snapping_tool/)
- **Lasso** – App and window organizer with layout saving
- **MacsyZones** – Snap windows to custom zones, similar to Windows PowerToys FancyZones [macoswm](https://macoswm.com)
- **Glide** – Recently updated window manager with snapping features [macoswm](https://macoswm.com)

***

## Tiling Window Managers (Power Users)

These automatically arrange all open windows into non-overlapping tiles — popular among developers and Linux users: [nek12](https://nek12.dev/blog/en/tiling-window-manager-macos-aerospace-vs-amethyst-vs-yabai/)

- **AeroSpace** – i3-inspired tiling WM for macOS; YAML config, fast workspace switching, stable and highly configurable; best for those coming from Linux [nek12](https://nek12.dev/blog/en/tiling-window-manager-macos-aerospace-vs-amethyst-vs-yabai/)
- **Yabai** – The oldest and most powerful tiling WM for macOS; supports BSP (binary space partitioning) and Fibonacci layouts; requires terminal configuration [nek12](https://nek12.dev/blog/en/tiling-window-manager-macos-aerospace-vs-amethyst-vs-yabai/)
- **Amethyst** – Lightweight automatic tiling with a GUI menu bar interface; xmonad-inspired; easiest tiling WM to get started with [nek12](https://nek12.dev/blog/en/tiling-window-manager-macos-aerospace-vs-amethyst-vs-yabai/)
- **Paneru** – Derived from Yabai's codebase; inspired by the niri WM [roguelazer](https://www.roguelazer.com/blog/2026-02-aerospace/)
- **Rift** – Another tiling-style window manager in active development [macoswm](https://macoswm.com)
- **PaperWM.spoon** – A Hammerspoon-based scrolling tiling WM inspired by PaperWM [macoswm](https://macoswm.com)
- **OmniWM** – Lightweight tiling WM [macoswm](https://macoswm.com)
- **Yashiki** – Tiling window manager recently updated [macoswm](https://macoswm.com)
- **HyprSpace** – A fork of AeroSpace [alper](https://alper.nl/posts/window-managers/)

***

## Scripting & Automation-Based

- **Hammerspoon** – Powerful Lua-scripted automation tool; can replicate any window management behavior via scripts
- **Keyboard Maestro** – Macro tool that can automate window positioning and resizing
- **Bunch** – Launch and arrange app sets with simple plain-text scripts

***

## Built-In macOS Features

macOS itself includes some native window management that has improved over time: [reddit](https://www.reddit.com/r/MacOS/comments/y89gcd/macos_window_management_tool/)

- **Stage Manager** – Introduced in macOS Ventura; groups active windows and recent apps
- **Split View** – Available since El Capitan; snap two apps side by side in full screen (hold the green button)
- **Window Tiling** – Introduced in macOS Sequoia; drag windows to edges/corners to tile them natively [youtube](https://www.youtube.com/watch?v=O4g3lLr21iE)

## Disclaimer

All code and "art"-work in this project is created 100% by AI.

I have created this projects for two reasons:

1. I wanted to use this tool. All current window management tools did not do what I wanted, so…

2. I have roughly 40 years of software development background, but I wanted to really see how a full project from zero to release would feel with AI-only. Every line of code in this project is written by Claude Code, with Perplexity.ai as research assistant. I did not write a single line of code myself (the occasional `NSLog()` for debugging doesn't count in my view).

On the first point: it really is the window management tool I always wanted.

On the second: it turns out that in the hands of someone with quite extensive software development background, these tools can do quite impressive things. I just would not want this tool wielded by someone who does not know what they are doing. This left the question in my mind of how in the future people are supposed to achieve this software development knowledge when no one is writing code anymore…
